数据库驻留连接共享

“数据库驻留连接共享”为跨客户机应用程序或进程的“专用”连接共享作好了准备。专用连接指的是专用服务器及其数据库会话。如果有多个客户机应用程序 (如中间层) 需要维持与数据库的永久连接，或者如果需要对服务器资源 (如内存) 进行优化，则使用“数据库驻留连接共享”。

在“数据库驻留连接共享”中，客户机连接到称为“连接中介”的提供共享功能的后台进程。

与专用服务器和共享服务器相对。
