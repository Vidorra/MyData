一般信息：跟踪

跟踪选项卡使您能够设置本计算机上的跟踪参数。如果本计算机是客户机，请使用客户机信息字段。如果本计算机是服务器，请使用服务器信息字段。

客户机跟踪保持对客户机执行的操作进行跟踪。服务器跟踪保持网络连接过程中的服务器执行的跟踪操作。

跟踪级别

从列表中，选择跟踪级别：

OFF - 尚未启用跟踪。OFF 是默认设置。

USER - 将跟踪设置为与用户相应的级别。跟踪以标识用户导致的错误条件。

ADMIN - 将跟踪设置为与数据库管理员相应的级别。跟踪以标识特定的安装问题。

SUPPORT - 将跟踪设置为与客户支持相应的级别。

跟踪文件可能会变得很大。Oracle Corporation 建议在不进行网络问题诊断时关闭跟踪功能。

跟踪目录

输入跟踪文件将写入的目录。当客户机和服务器跟踪文件传送有关客户机或服务器的信息时，这两种信息都将写入本地计算机中。默认值为 $ORACLE_ HOME/network/trace (在 UNIX 上) 和 ORACLE_HOME\network\trace (在 Windows 平台上)。

跟踪文件

输入跟踪文件名。默认情况下，客户机跟踪文件名为 sqlnet.trc，服务器跟踪文件名为 svr_pid..trc。

唯一跟踪文件名称

选择此选项以创建每个跟踪文件的唯一名称。在每个跟踪文件名后追加进程标识符以便多个跟踪文件可以共存。如果您不想创建唯一的跟踪文件，并希望所有的跟踪信息都存储在相同的跟踪文件中，请清除此选项。
