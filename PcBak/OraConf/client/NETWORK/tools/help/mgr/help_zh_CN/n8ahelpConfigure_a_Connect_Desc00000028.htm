配置连接到 Oracle Connection Manager 的连接描述符

要通过 Oracle Connection Manager 将客户机路由至数据库服务器，请配置 tnsnames.ora 文件或带有指定 Oracle Connection Manager 的协议地址的连接描述符的目录服务器。客户机可以使用此地址连接到 Oracle Connection Manager 所在的计算机。

要配置 Oracle Connection Manager 的协议地址，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录/本地 > 服务命名。

  2. 从工具栏中选择加号 (+)，或从菜单栏中选择编辑 > 创建。将显示“网络服务名向导”的“欢迎使用”页。

  3. 在网络服务名字段中输入任意名称。

  4. 选择下一步。此时将显示“协议”页。

  5. 选择 Oracle Connection Manager 被配置在其上监听的协议。默认情况下，该协议为 TCP/IP。

  6. 选择下一步。此时将显示“协议设置”页。

  7. 在提供的字段中为所选协议输入相应的协议参数。如果正在使用 TCP/IP，则默认端口是 1630。

  8. 选择下一步。此时将显示“服务”页。

  9. 如果目标服务是 Oracle8i 或更高版本的服务，请选择 Oracle8i 或更高版本，然后在服务名字段中输入服务名。如果目标服务是 Oracle8 或 Oracle7，请选择 Oracle8 或更低版本，然后在数据库 SID 字段中输入 Oracle 系统标识符。

  10. 选择下一步。此时将显示“服务”页。

  11. 单击完成保存配置并关闭“网络服务名向导”。

  12. 选择文件 > 保存网络配置。

  13. 新的网络服务名将添加到“服务命名”文件夹。

有关全面启用 Oracle Connection Manager 功能的详细信息，请参阅 Oracle10i Net Administrator's Guide 中的第 11 章“Configuring and Administering Oracle Connection Manager”。
