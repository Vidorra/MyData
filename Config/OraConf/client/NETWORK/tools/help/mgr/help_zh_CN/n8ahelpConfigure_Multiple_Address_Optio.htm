配置多个地址选项

当数据库服务可被多个监听程序的协议地址访问时，指定协议地址的使用顺序是非常重要的。例如，可以随机选择协议地址，或者按顺序尝试。

当为网络服务名或数据库服务配置了多个协议地址时，则以下选项是可配置的：

  • 连接时故障转移用来选择第一个协议地址，并在第一个协议地址失败时切换到下一个协议地址。

  • 客户机负载平衡用来随机选择协议地址。

  • 源路由选择用来按顺序使用所有协议地址。

要配置多个地址选项，请执行以下操作：

  1. 在导航器窗格中，展开 Oracle 网络配置 > 目录 > 本地或服务命名。

  2. 选择网络服务名或数据库服务。右侧窗格显示当前目标服务和地址列表。

  3. 在地址配置框中，选择高级。此时将显示地址列表选项对话框。

  4. 如果此配置适用于 Oracle9i 或 Oracle8i 客户机，请清除使用与 Net8 8.0 客户机兼容的选项。

  5. 选择协议地址列表的相应选项，然后选择确定。

  6. 在右侧窗格中，选择应用。

  7. 如果要对本地文件夹进行上述更改，请选择文件 > 保存网络配置。
